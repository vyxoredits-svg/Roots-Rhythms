Roots & Rhythms Admin Setup

This folder is just a guide. The working admin panel lives in your deployed Google Apps Script web app.

What you are building

The public site stays static. Posts are stored in Google Sheets. Admins log into the Apps Script web app and publish posts there.

Files to use

  1. google-apps-script/Code.gs for the backend logic
  2. google-apps-script/Admin.html for the admin page
  3. script.js for loading public posts

What to do next

  1. Create a Google Sheet for the posts.
  2. Open Extensions → Apps Script.
  3. Paste in the code from google-apps-script/Code.gs.
  4. Add the HTML from google-apps-script/Admin.html as a new HTML file named Admin.
  5. Set the Apps Script property ADMIN_PIN.
  6. Optional but recommended: set SHEET_ID to the spreadsheet ID.
  7. Deploy as a Web App.
  8. Put that web app URL into script.js.

Public site note

The homepage loads posts from the Apps Script endpoint using JSONP, so the UI on the site itself does not need to change.

Open the admin web app after deployment:

https://script.google.com/macros/s/YOUR_DEPLOYED_ID/exec?view=admin
